## 1. Overview

**Radiance Copilot** is a desktop application built for CBSA officers to manage customs seizure paperwork. It runs locally on each officer's Windows PC — no internet connection or server is required to use the application itself.

All shared data — case folders, generated PDFs, and form templates — lives on a shared network drive mapped as **G:**. This means:

- Every officer works from the same set of current templates.
- Case files are accessible to any authorized officer on the network.
- The application on each PC is a lightweight launcher; the G: drive is the single source of truth for all case data.

**Key principle:** The app runs on the PC. The work product lives on the G: drive.

## 2. G: Drive Folder Structure

The G: drive is organized into two main areas: a shared **Radiance** root folder containing templates and configuration, and a **cases** subfolder where individual seizure case folders are created.

```
G:\
└── Radiance\
    ├── templates\
    │   ├── SAISIE À FAIRE_francompact 2025 - TEMPLATE.pdf
    │   ├── K138 Seizure Cannabis-Stupefiant - TEMPLATE.pdf
    │   ├── K138 Seizure Knives-Arms - TEMPLATE.pdf
    │   ├── K138 Stupefiant-Others - TEMPLATE.pdf
    │   ├── Agenda - TEMPLATE.pdf
    │   ├── Saisie d'intérêt - TEMPLATE.pdf
    │   ├── k138_note_cannabis.txt
    │   ├── k138_note_arms.txt
    │   └── k138_note_other.txt
    │
    └── cases\
        ├── 19747 2026-02-11 W0000604 - Cannabis\
        │   ├── 20260211_SAISIE_scan.pdf           ← BSO deposits completed SAISIE here
        │   ├── Agenda_19747.pdf                   ← generated by BSO
        │   ├── K138_20260211_SAISIE_scan.pdf      ← generated by Clerk
        │   ├── Saisie_Interet_19747.pdf           ← generated by Clerk or BSO
        │   ├── barcode_19747.png                  ← generated automatically
        │   └── .extracted_data\                   ← hidden folder (see Section 4)
        │       └── .19747 2026-02-11 W0000604 - Cannabis\
        │           └── values_latest.json
        │
        ├── 19800 2026-02-15 V0000712 - Knives\
        │   └── ...
        │
        └── 19850 2026-02-18 T0000891 - Others\
            └── ...
```

### Naming Convention for Case Folders

Case folders should follow a consistent naming pattern so they are easy to find and sort:

```
<Badge#> <Date YYYY-MM-DD> <WarehouseRef> - <SeizureType>
```

Example: `19747 2026-02-11 W0000604 - Cannabis`

This format makes case folders sort chronologically and lets any officer instantly identify the case type at a glance.

## 3. Templates on the G: Drive

All template PDFs and supporting notice text files live in `G:\Radiance\templates\`. This is a **read-only reference area** — the application reads from it but never writes output files into it.

### Why Templates Live on the G: Drive

- **Automatic updates.** When IT or a supervisor updates a template (for example, a revised K138 form), every officer gets the new version immediately the next time they process a case. No one needs to manually copy files to their PC.
- **Consistency.** Everyone is always working from the same approved version of every form.
- **No manual maintenance.** Officers never need to know where templates are stored or whether their copy is current.

### Template Files Required

| File | Purpose |
|---|---|
| `SAISIE À FAIRE_... - TEMPLATE.pdf` | Blank reference SAISIE form used for data extraction |
| `K138 Seizure Cannabis-... - TEMPLATE.pdf` | Blank K138 for cannabis seizures |
| `K138 Seizure Knives-Arms - TEMPLATE.pdf` | Blank K138 for knives/arms seizures |
| `K138 Stupefiant-Others - TEMPLATE.pdf` | Blank K138 for all other seizure types |
| `Agenda - TEMPLATE.pdf` | Blank Agenda form |
| `Saisie d'intérêt - TEMPLATE.pdf` | Blank Saisie d'intérêt form |
| `k138_note_cannabis.txt` | Legal notice text appended to cannabis K138 forms |
| `k138_note_arms.txt` | Legal notice text appended to knives/arms K138 forms |
| `k138_note_other.txt` | Legal notice text appended to other K138 forms |

### Updating Templates

To update a template, IT staff or a designated supervisor:

1. Opens `G:\Radiance\templates\` directly.
2. Replaces the old template file with the new version, keeping the same filename.
3. No other action is needed — all officers will automatically use the new template on their next case.

Notice text files (`.txt`) can be edited in Notepad at any time to update legal language without touching the PDF templates or the application itself.

## 4. What Saves Where — Local vs. G: Drive

Radiance Copilot separates personal settings (which stay on the local machine) from case data (which goes to the G: drive). This separation is intentional and important.

### What Saves LOCALLY (on each officer's PC)

**File:** `radiance_copilot.cfg`
**Location:** The same folder as the application executable, on the local PC.

This file stores only personal preferences for that one workstation:

| Setting | Description | Example |
|---|---|---|
| `profile_role` | The officer's role in the app | `BSO`, `Clerk`, or `Supervisor` |
| `profile_name` | The officer's name | `Smith` |
| `badge_number` | The officer's CBSA badge number | `19747` |
| `templates_folder` | Path to the templates folder on G: drive | `G:\Radiance\templates` |
| `saisie_folder` | Last case folder browsed to (for convenience) | `G:\Radiance\cases\19747 2026-02-11...` |

**This file is configured once** when the app is first set up on each PC. After that, it loads automatically on startup. Officers do not need to reconfigure anything between sessions.

The config file does not contain case data or sensitive seizure information — only paths and identity settings.

### What Saves to the G: DRIVE (shared, in the case folder)

Everything generated during case processing is written directly to the case folder on the G: drive:

| File | Description |
|---|---|
| Completed SAISIE PDF | The scanned/filled SAISIE form deposited by the BSO |
| `Agenda_<case>.pdf` | The filled Agenda PDF generated by the BSO |
| `K138_<saisie_stem>.pdf` | The filled K138 seizure notice generated by the Clerk |
| `Saisie_Interet_<case>.pdf` | The filled Saisie d'intérêt form |
| `barcode_<case>.png` | Barcode image generated for the case |
| `.extracted_data\` (hidden folder) | Contains `values_latest.json` — the extracted data from the SAISIE form, used to pre-fill subsequent documents |

The `.extracted_data` folder is hidden (its name starts with a dot) to keep the case folder tidy. Officers do not need to open or interact with this folder — it is used automatically by the application.

**`values_latest.json`** is the key handoff file between officers. When the BSO processes a SAISIE and generates an Agenda, the extracted data is saved to this JSON file. When the Clerk later opens the same case folder to generate the K138, the application reads from this file automatically — the Clerk does not need to re-enter any information.

## 5. Role-Based Workflow on the Shared Drive

Radiance Copilot has three officer roles. Each role controls which tabs and actions are available in the application. All three roles work from the same G: drive case folders.

### BSO (Border Services Officer)

The BSO is the first officer to interact with a seizure case.

**Typical workflow:**

1. **Create the case folder** on the G: drive under `G:\Radiance\cases\`, using the standard naming convention.
2. **Copy or scan the completed SAISIE PDF** into the case folder.
3. **Open Radiance Copilot** on their workstation. The app is already configured to point to `G:\Radiance\templates\`.
4. **Select the SAISIE PDF** from the case folder on G: drive using the Browse button (the app remembers the last folder used).
5. **Select the seizure type** (Cannabis, Knives, or Others).
6. **Click "Process PDF."** The app:
   - Extracts all data fields from the SAISIE form.
   - Saves extracted values to `.extracted_data\values_latest.json` in the case folder.
   - Generates the filled **Agenda PDF** and saves it to the case folder on G: drive.
   - Generates a **barcode PNG** and saves it to the case folder.
7. The case folder on G: drive now contains everything the Clerk needs.

The BSO does not need to email files or copy anything — the Clerk simply opens the same folder on the G: drive.

---

### Clerk

The Clerk's role is to generate the formal K138 seizure notice after the BSO has processed the SAISIE.

**Typical workflow:**

1. **Open Radiance Copilot** — the app opens in Clerk mode (as saved in local config).
2. **Click "Select Agenda"** (or "Select Case Folder") and browse to the relevant case folder on `G:\Radiance\cases\`.
3. The application reads `values_latest.json` from the hidden `.extracted_data` folder automatically — no data entry needed.
4. **Select the form type** if not already pre-filled.
5. **Click "Generate K138."** The app:
   - Reads the K138 template from `G:\Radiance\templates\`.
   - Fills in all case fields automatically.
   - Appends the appropriate legal notice text from the `.txt` file.
   - Saves the completed **K138 PDF** to the case folder on G: drive.
6. Optionally, generates the **Saisie d'intérêt** form from the same data.

The Clerk never needs to re-type information that the BSO already captured. All data flows automatically through the shared `values_latest.json` file.

---

### Supervisor

The Supervisor role has full access to all application tabs and can open any case folder on the G: drive.

**Typical use:**

- Reviewing completed case folders (Agenda, K138, Saisie d'intérêt) before signing off.
- Re-generating any document if a correction is needed.
- Browsing across multiple case folders to monitor workload.
- Updating notice text files in `G:\Radiance\templates\` if legal language changes.

Supervisors do not require special file system permissions beyond read/write access to `G:\Radiance\` — the same access all officers have.

## 6. Installation

### What Gets Installed on Each PC

Radiance Copilot is distributed as a single self-contained Windows executable (`R-Copilot.exe`), built with PyInstaller. No Python installation is required on officer workstations.

### Option A — Single Executable (Recommended)

1. IT places `R-Copilot.exe` in a stable local folder on each PC, for example:
   ```
   C:\Program Files\RadianceCopilot\R-Copilot.exe
   ```
2. A shortcut is created on the officer's Desktop pointing to that executable.
3. On **first launch**, the officer:
   - Enters their name, badge number, and selects their role (BSO / Clerk / Supervisor).
   - Clicks "Browse" next to the Templates Folder field and navigates to `G:\Radiance\templates\`.
   - The application saves these settings to `radiance_copilot.cfg` in the same folder as the executable.
4. From that point on, the officer simply double-clicks the shortcut — no further configuration is needed.

### Option B — Batch Script Launcher

For environments where distributing an executable is not preferred, IT can place the Python source files on a shared location and provide each officer with a `.bat` launcher:

```bat
@echo off
cd /d "G:\Radiance\app\"
python saisie_a_faire_extractor.py
pause
```

This requires Python and the required libraries (`PyMuPDF`, `reportlab`, `PyPDF2`, `tkinterdnd2`) to be installed on each workstation.

### First-Time Configuration Per Machine

The only step that needs to happen once per machine is pointing the app to the G: drive:

| Setting | Value to Enter |
|---|---|
| Templates Folder | `G:\Radiance\templates` |
| Role | Select appropriate role for that officer |
| Badge Number | Officer's CBSA badge number |

After saving, these values persist in `radiance_copilot.cfg` and are loaded automatically on every subsequent launch.

### Updating the Application

When a new version of `R-Copilot.exe` is released, IT replaces the executable on each machine (or on the shared location if using the batch launcher). The `radiance_copilot.cfg` file is preserved — officers do not need to reconfigure anything after an update.

## 7. Practical Considerations

### File Locking — One Person Per Case at a Time

Windows does not prevent two people from opening the same folder simultaneously, but writing to the same PDF at the same time can cause file corruption or errors. The following practice should be established:

- **Only one officer should actively process a given case at any one time.** In practice, this is natural: the BSO finishes their steps before the Clerk begins.
- If two officers attempt to write to the same case folder simultaneously, the second officer will receive a "Permission denied" or file-in-use error. The fix is simple: close any open PDFs and try again.
- For offices with high case volume, a simple whiteboard or shared log noting which cases are "in progress" is sufficient. No additional software locking mechanism is required.

### Offline Access — What Happens If the G: Drive Is Unavailable

If the G: drive is temporarily disconnected (network outage, VPN issue, mapped drive not connected):

- The application will **launch normally**, since it runs from the local PC.
- Officers **will not be able to open or save case files**, because all case data lives on the G: drive. The Browse dialog will show an error or empty folder.
- Officers **can still view previously opened PDFs** if those files were already open in a PDF viewer before the connection dropped.
- **No data is lost.** All previously saved work remains intact on the G: drive and will be accessible once the connection is restored.

**Recommended practice:** If an officer knows they will be working without a network connection (for example, at a remote port of entry), they should copy the relevant case folder to their local desktop before going offline, and copy it back when reconnected.

### Keeping Templates Up to Date

Because templates live on the G: drive, keeping them current is simple:

- **IT or a designated supervisor** is responsible for the `G:\Radiance\templates\` folder.
- When a form is revised (for example, a new version of the K138), the old template PDF is replaced with the new one, keeping the same filename.
- **No officer intervention is needed.** The next time any officer processes a case, they automatically use the new template.
- Notice text files (`k138_note_cannabis.txt`, etc.) can be edited in any text editor at any time to update legal language. Changes take effect immediately on the next case processed.

### Permissions Summary

| Folder | Officers | IT/Supervisor |
|---|---|---|
| `G:\Radiance\templates\` | Read-only | Read/Write |
| `G:\Radiance\cases\` | Read/Write | Read/Write |
| Local `radiance_copilot.cfg` | Read/Write (own PC only) | Read (for support) |

Restricting officers to read-only access on the templates folder prevents accidental deletion or modification of master templates.

### Backup

- `G:\Radiance\cases\` should be included in the standard network drive backup schedule.
- `G:\Radiance\templates\` should also be backed up, with version history retained if possible, so that previous template versions can be restored if needed.
- Local `radiance_copilot.cfg` files do not need to be backed up — they contain only personal settings that can be re-entered in under two minutes if a PC is replaced.
